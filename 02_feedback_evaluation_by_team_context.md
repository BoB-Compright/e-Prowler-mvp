## 종합평

**🟡 조건부 가능 — 범위를 "서버 자동 수집 + AI 분석 + 대시보드" 3가지로 명확히 잘라서 집중하면 7/15까지 완주 가능**

**핵심 메시지:** 시간이 1인당 약 20시간이므로, 꼭 필요한 3가지에만 집중하고 나머지는 "하지 않을 것"으로 명시해야 함.

## 팀 & 시간 분석

### 팀 구성
- **Claude Code + git 경험:** 1명 (병목 리스크)
- **초보자:** 2명 (DevSecOps/인프라 배경)
- **작업 방식:** 비동기 + 본업 병행

### 실제 개발 시간
```
3시간/일 × 20일 ÷ 3명 = 1인당 약 20시간
```
→ 매우 짧음. 기술 선택·셋업·개발을 동시에 하기 부족

## ✅ 잘한 점 (살릴 강점)

### 1. 명확한 핵심 설정
- "서버 자동 수집"이 진짜 핵심이라는 걸 알고 있음
- 이 1개 기능만 완성해도 가치 있는 프로토타입 가능

### 2. 환경 준비
- 도커 이미지 준비 (배포 준비 완료)
- Claude API, 외부 API 할당량 확보
- 불필요한 장애물 없음

### 3. 팀 배경
- DevSecOps/인프라 경험 → OS 수집·분석·취약점 이해 수월
- 보안 마인드 기본 갖춤

## ⚠️ 부족한 점 (완주를 위협하는 것)

### 1. 팀원 협업 병목
**리스크:** Claude Code + git 경험 1명이 초보 2명을 리드하는 구조
- 비동기 작업 → 코드 리뷰 지연
- 본업 병행 → 응답 시간 느림
- PR 충돌 해결, 코드 통합이 복잡해질 수 있음

**대응:**
- 명확한 역할 분담 (백엔드 / AI 분석 / 프론트엔드)
- 일일 스탠드업 또는 비동기 체크인
- 경험자 1명의 코드 리뷰/통합 시간 확보

### 2. 시간 부족
**리스크:** 1인당 20시간은 기술 선택·셋업·개발을 동시에 하기 빠듯함
- 백엔드(OS 수집 + AI 분석): 20시간
- 프론트엔드(대시보드): 20시간
- DevOps(배포·통합): 20시간

**대응:**
- 기술 스택을 **내일 중 결정** (선택 시간 제거)
- 프로토타입 우선 (완성도 낮아도 작동하는 것)
- 범위를 엄격히 제한 (3가지만)

### 3. 기술 스택 미결정
**리스크:** 선택 지연 → 1-2일 시간 낭비
- FastAPI vs Flask
- PostgreSQL vs MySQL
- React vs Vue

**대응:**
- 팀이 가장 빠르게 할 수 있는 것 선택 (학습곡선 아님)
- 추천: FastAPI(Python FastAPI는 보안 기본값 좋음) + PostgreSQL(많이 쓰임) + React(초보 자료 많음)

## 🚨 주의사항

1. **범위를 절대 늘리지 말 것**
   - 지금도 시간이 부족한데, 기능이 추가되면 완주 불가
   - 기능 요청이 오면 "7/15 이후 v1.1에" 로 연기

2. **기술 선택은 내일까지**
   - 선택 지연 = 개발 시간 손실
   - 완벽한 선택보다 **빠른 선택**이 중요

3. **프로토타입 마인드셋**
   - 완벽하게 하려고 하면 못 함
   - "작동하는 것"이 목표
   - 코드 품질은 최소한 수준으로

4. **CVE DB는 나중에**
   - 필수 아님. MVP에는 "수집 + 분석 + 표시"만 있어도 됨
   - 정적 CVE 데이터로 테스트해도 OK
