# 대시보드 시인성 개선 설계

날짜: 2026-07-11
상태: 승인됨

## 배경 / 문제

현재 대시보드(`src/app/page.tsx`)는 "자산 보안 상태" 테이블이 등록된 **전체 자산을 무제한으로 나열**해서, 자산이 많아지면 메인 화면이 수직으로 계속 길어진다. 또한 KPI 숫자 타일 외에 대시보드다운 시각화 요소(차트, 종합 점수)가 없고, 최근 활동 피드는 하단에 있어 스크롤해야 보인다.

## 목표

- 자산 수와 무관하게 대시보드 페이지 길이를 일정하게 유지한다.
- 한눈에 보안 수준을 파악할 수 있는 시각화(종합 점수 게이지, 상태 분포 도넛)를 추가한다.
- 최근 활동 피드를 항상 보이는 위치(사이드 컬럼)로 옮기고 내용을 강화한다.

## 비목표

- 점수 추이(시계열) 그래프 — 이번 범위에서 제외.
- /assets, /projects 등 다른 페이지 변경.
- 외부 차트 라이브러리 도입 — SVG 직접 렌더링만 사용.

## 레이아웃

```
┌────────────────────────────────────────────────┬──────────────┐
│ KPI 타일 4개 (총 자산 / 취약 / CVE / 스케줄)      │              │
├───────────────┬────────────────────────────────┤  최근 활동    │
│ 종합 보안 점수 │  자산 상태 분포 도넛             │  피드        │
│  (게이지)      │  (양호/검토/취약/미점검)         │  (최근 10건) │
├───────────────┴────────────────────────────────┤              │
│ 위험 자산 TOP 5        [전체 자산 보기 →]        │              │
├────────────────────────────────────────────────┤              │
│ 고위험 CVE TOP 5 (기존 유지)                     │              │
└────────────────────────────────────────────────┴──────────────┘
```

- `lg` 브레이크포인트 이상: 메인 컬럼(2/3) + 사이드 컬럼(1/3) 그리드.
- 모바일: 세로 스택 (KPI → 점수 → 도넛 → TOP5 → CVE → 피드).
- 기존 자산 전체 테이블은 제거하고 TOP 5 + "전체 자산 보기 →"(/assets) 링크로 대체.

## 컴포넌트 설계

### 1. 종합 보안 점수 (SecurityScoreGauge)

- 순수 함수 `computeSecurityScore(input): { score: number; grade: Grade }`를
  `src/lib/dashboard/securityScore.ts`에 구현하고 단위 테스트를 붙인다.
- 산정식 (0~100, 100에서 감점):
  - 취약 자산 비율 × 40 (최대 −40)
  - min(30, 전체 Critical/High 점검 항목 수 × 2) (최대 −30)
  - min(30, 미해결 Critical/High CVE 수 × 3) (최대 −30)
  - 미점검 자산 비율 × 10 (최대 −10, 커버리지 감점)
  - 하한 0으로 클램프. 자산이 0개면 점수 대신 빈 상태 표시.
- 구간: 90+ 안전(초록) / 70~89 주의(노랑) / 40~69 경고(주황) / <40 위험(빨강).
- 표시: SVG 원형(아크) 게이지, 중앙에 점수·/100·구간 라벨.

### 2. 상태 분포 도넛 (AssetStatusDonut)

- 자산별 최근 점검 결과를 양호/검토/취약/진행 중/미점검으로 집계.
- 집계 로직은 `src/lib/dashboard/statusDistribution.ts` 순수 함수로 분리, 단위 테스트.
- SVG 도넛 + 우측 범례(라벨·건수). 구현 시 dataviz 스킬의 색·접근성 기준을 따른다.

### 3. 위험 자산 TOP 5 (RiskyAssetsCard)

- 정렬: 상태(취약 > 검토 > 미점검 > 양호) → Critical/High 점검 항목 수 내림차순 → 미해결 CVE 수 내림차순.
- 행: 자산명(/assets/:id 링크) · 상태 배지 · C/H 건수 · CVE 건수.
- 카드 하단 "전체 자산 보기 →" 링크(/assets).
- 정렬 로직도 순수 함수로 분리해 테스트.

### 4. 최근 활동 피드 (ActivityFeed, 사이드)

- 이벤트 소스:
  - 점검 완료: 결과 요약 포함 ("취약 3건 · 검토 1건" — riskSummary 활용)
  - 점검 시작/진행 중, 실패, 취소
  - 자산 등록 (assets.created_at에서 유도)
- 두 소스를 시간 역순으로 병합해 최근 10건 표시.
- 상대 시간 표시("5분 전", "1시간 전", 24시간 이후는 날짜) — 서버 렌더 시각 기준.
- 진행 중 점검이 있으면 기존 AutoRefresh 패턴으로 주기 갱신.

## 유지되는 것

- KPI 타일 4개 (총 자산 / 취약 자산 / 미해결 CVE / 활성 스케줄)
- 고위험 CVE TOP 5 카드
- 하단 LocalImageFallbackForm
- 자산 0개일 때의 빈 상태 안내

## 에러/엣지 케이스

- 자산 0개: 기존 빈 상태 카드 유지 (차트 렌더 안 함).
- 점검 이력이 하나도 없는 경우: 점수 게이지는 커버리지 감점만 반영된 상태로 표시, 도넛은 전부 "미점검".
- CVSS/severity 누락 CVE: 기존 대시보드와 동일하게 severity 문자열 폴백.

## 테스트 전략

- `securityScore`, `statusDistribution`, TOP 5 정렬 — 순수 함수 단위 테스트 (경계값: 자산 0, 전부 양호, 전부 취약, 감점 상한 클램프).
- 페이지 수준은 기존 프로젝트 패턴(로직을 lib으로 추출해 테스트)을 따르고, 구현 후 실제 데이터로 브라우저 확인(verify)을 거친다.
