# 리스킨 생략 기능 백로그

날짜: 2026-07-09
관련 작업: Stitch "Kinetic Security System" 전체 리스킨 ([`docs/superpowers/specs/2026-07-09-stitch-kinetic-reskin-design.md`](2026-07-09-stitch-kinetic-reskin-design.md), [`docs/superpowers/plans/2026-07-09-stitch-kinetic-reskin.md`](../plans/2026-07-09-stitch-kinetic-reskin.md))

이 문서는 **리스킨에서 의도적으로 생략된 목업(Stitch mockup) 전용 기능을 향후 기능 개발 후보로 넘기는 백로그**다. 리스킨은 "기능·데이터 바인딩·API는 일절 건드리지 않는 순수 UI 리스킨"이라는 제약 아래 진행되었고, 각 화면 태스크(Task 3~10)는 목업에는 있지만 실제 백엔드 기능·데이터가 없는 요소를 구현하지 않고 각 태스크 리포트의 "생략한 목업 전용 기능" 섹션에 기록했다. 이 문서는 그 기록 전체를 화면별로 취합한 것이다 — 신규 기능 개발 시 후보 목록으로 참고한다.

각 항목 형식: **기능** — 생략 사유 (필요 선행 작업)

## 앱 셸 (사이드바·헤더) — Task 3

- **알림(Notification) 아이콘/배지** (유틸 헤더 우측) — 대응하는 알림 기능 자체가 없음 (알림 시스템/이벤트 소스 설계 필요)
- **사용자 프로필 블록** (유틸 헤더 우측 끝, 프로필 사진 + 사용자명) — 인증 통합 필요 (로그인/세션 사용자 정보 확보 필요)
- **사이드바 하단 사용자 정보 + 로그아웃 버튼** — 세션 관리 필요 (인증/세션 인프라 필요)
- **사이드바 토글 버튼** (모바일 헤더 햄버거 메뉴) — 모바일 사이드바 열기 상태 관리 미구현 (클라이언트 상태 관리 도입 필요)
- **검색 바** (유틸 헤더 중앙) — 검색 대상·인덱스가 없음 (검색 기능 자체 개발 필요)

## 대시보드 (`/`) — Task 4

- **상단 액션 버튼**: "템플릿 다운로드" / "보고서 내보내기" / "전체 스캔 실행" — 대응 기능이 없어 구현하지 않음 (다운로드/내보내기/일괄 스캔 API 필요)
- **KPI 타일 트렌드 화살표·증감률** (예: "+2 since last scan") — 시계열/이력 데이터가 없는 장식적 가짜 지표라 제외 (지표 스냅샷 히스토리 저장 필요)
- **자산 보안 상태 테이블 "Actions"(⋮ 더보기 메뉴) 컬럼** — 대응하는 실제 액션이 없음 (행별 액션 기능 정의 필요)
- **고위험 CVE 카드의 원형 CVSS 게이지(뱃지)** — 현재 텍스트(`CVSS x.x`)로 대체, 원형 게이지는 시각 장식 (디자인 결정 필요 시 게이지 컴포넌트 도입)
- **최근 활동 피드의 타임라인 커넥터 및 감사로그성 활동** (예: "Admin User가 정책을 업데이트") — 실데이터 소스가 없음(현재 피드는 run 이력만 존재) (감사 로그/이벤트 소싱 필요)
- **로컬 이미지 스캔 폴백 카드의 "Select Image Archive"/"View Docs" 버튼, 배경 장식 아이콘** — 실제 기능과 무관한 장식/미구현 기능 (기존 select+제출 폼으로 대체 유지)

## 자산 관리 (`/assets`, `/assets/[id]`, `/assets/new`, `/assets/upload`) — Task 5

- **자산 목록 테이블의 상태·위험도 컬럼** (목업: `assets_list.html` Pass/Fail/Review) — 자산별 최신 점검 결과를 표시하려면 `listRuns`/`getRunRiskSummary`를 목록 페이지에 새로 연결해야 해 기능 변경 범위를 벗어남 (대시보드 `/`의 "자산 보안 상태" 테이블이 현재 이 역할을 대체 수행 중)
- **자산 목록 행별 아이콘 액션(실행/편집/삭제), 검색창, "Showing N of M" 카운터** — 대응 기능이 없음 (검색/인라인 액션 기능 설계 필요)
- **자산 상세의 드로어(오버레이 패널) 레이아웃과 탭(감지된 CVE/스캔 설정/점검 이력) 내비게이션, "편집" 버튼** — 실제 화면은 전체 페이지 구조이고 편집 기능 자체가 없음 (드로어 전환 시 클라이언트 라우팅/상태 구조 변경 필요, 편집 API 필요)
- **자산 상세의 "OS"·"담당자" 필드** — 현재 `Asset` 데이터 모델에 해당 필드가 없음 (데이터 모델 확장 필요)
- **신규 자산 등록의 라디오 버튼형 인증방식 선택** (현재 select 유지), CSV 템플릿 다운로드 링크 위치 등 — 기존 폼 필드 구성·컨트롤 타입을 그대로 유지 (UX 변경 결정 시 별도 논의)

## 점검 이력·진행 현황 (`/runs`, `/runs/[id]`, `/runs/batch/[batchId]`) — Task 6

- **좌측 사이드바의 "Start New Scan" 버튼, 별도 nav 구조** — 이미 공용 `AppSidebar`로 대체됨 (추가 작업 불필요)
- **상단 "스캔 취소" 버튼** — 현재 파이프라인에 취소 API/기능이 없음 (점검 취소 API 필요)
- **"실시간 모니터링" radar 라벨, 고정 자산 배지**(예: `PROD-DB-CLUSTER-01`) — 기존 화면은 `run.repoUrl` 표시로 대체(레포/서버 공용 구조라 목업처럼 고정 자산명 배지를 넣지 않음) (해당 없음, 설계상 대체)
- **우측 "대상 자산 정보" 사이드 패널** (자산유형/환경 태그) — 자산 상세는 이미 `/assets/[id]`에 있어 중복 (해당 없음)
- **"예상 남은 시간"(hourglass, `02:15`) 위젯** — 남은 시간 추정 데이터가 파이프라인에 없음 (점검 단계별 소요 시간 추정 로직 필요)
- **로그 헤더의 "수집률: 45%" 배지** — 해당 진행률 지표가 실데이터로 존재하지 않음 (로그 수집 진행률 산출 로직 필요)
- **AI 단계의 `aiGlow`/`sparkleTwinkle`/`sparkleFloat` 장식 애니메이션** — 실데이터가 뒷받침하지 않는 순수 장식이라 제외, 카드 강조(`border-primary/40 bg-primary/5`)와 실제 `isAiStage` 판정 기반 배지만 유지 (해당 없음, 디자인 결정)

## 점검 리포트 (`/runs/[id]/report`) — Task 7

- **"보고서 내보내기" 버튼** — 실제 export(PDF/CSV) 기능이 없음 (export API 필요, 생기면 Secondary 버튼 레시피로 추가 가능)
- **"재스캔" 버튼** — report 화면에서 즉시 재스캔을 트리거하는 기능이 없음(자산 상세의 `StartScanButton`/`FleetScanButton`과 다른 컨텍스트) (재스캔 트리거 API/UX 설계 필요)
- **실패 항목을 스택형 카드로 전부 펼쳐 보여주는 "Claude AI 취약점 분석 (Failed Items)" 레이아웃** — 기존 카테고리/상태 필터 + 좌측 목록 + 우측 상세(master-detail) 인터랙션이 이미 동일 정보를 필터링 가능하게 제공 중이라 유지, 목업 레이아웃으로 교체 시 필터/탐색 기능 손실 (UX 결정 필요 시 별도 기능 티켓)

## 카탈로그 (`/catalog`) — Task 8

- **사이드 필터 패널** (Categories 체크박스: All Frameworks/Container/KISA Unix/KISA Web, Status 체크박스: Automated/Manual) — 현재 `/catalog`에는 필터링 기능 자체가 없음 (쿼리 파라미터/서버 필터링 로직 설계 필요)
- **검색창** ("Search catalog...") — 카탈로그 항목 검색 기능 없음 (검색 기능 개발 필요)
- **Export 버튼** — 카탈로그 내보내기 기능 없음 (export API 필요)
- **뷰 토글**(List/Grid 아이콘) — 뷰 전환 기능 없음 (그리드 뷰 레이아웃 설계 필요)
- **행 클릭 시 상세 이동 + hover chevron** — 카탈로그 항목 상세 페이지/드릴다운 기능 없음 (상세 페이지 라우트 신설 필요)
- **"Showing N items" 툴바 카운터 바** — 카테고리별 카운트로 대체(이미 구현됨), 목업처럼 별도 툴바는 두지 않음 (해당 없음, 설계상 대체)

## 프로젝트 관리 (`/projects`, `/projects/[id]`) — Task 9

- **검색창** ("Search projects...", 상단/모바일) — 서버 사이드 필터링 로직이 없음 (검색 기능 개발 필요)
- **KPI 벤토 그리드** (Total Projects / Active Scans 카드, Global Fleet Status 배너) — 집계 API가 없음 (집계 API 설계 필요)
- **테이블 목록 + 우측 슬라이드 드로어(Detail Drawer)** (프로젝트명, Critical Risk 배지, Quick Stats, Grouped Assets 아코디언, Export Report/Run Fleet Scan 푸터 액션) — 드로어 전환은 클라이언트 상태 관리·라우팅 방식 자체를 바꿔야 해 마크업만 변경하는 리스킨 범위를 벗어남, 현재 `/projects/[id]` 전체 페이지 내비게이션 구조 유지 (라우팅 아키텍처 변경 결정 필요)
- **프로젝트 목록 행 내 "Fleet Scan" 버튼 및 "Share" 아이콘 버튼** (테이블 액션 컬럼) — 목록에서 바로 일괄 점검 실행/공유 모달을 여는 액션. 현재는 상세 페이지 진입 후에만 `FleetScanButton`/`ShareLinkPanel` 사용 가능 (목록 인라인 액션 UX 설계 필요)
- **"View All Projects" 하단 페이지네이션 링크** — 프로젝트 수 제한된 미리보기 UI. 현재는 전체 목록을 항상 보여주므로 불필요 (해당 없음)
- **공유 설정 모달의 "External Sharing" on/off 토글** — 공유 링크 활성/비활성 상태가 백엔드에 없음 (공유 링크 활성화 상태 필드 및 API 필요)
- **공유 설정의 "Revoke Access"(위험) 버튼** — 링크 폐기 API가 없음 (공유 링크 폐기 API 필요)
- **Vuln Score / 자산 그룹(Grouped Assets) 아코디언, 자산별 CVSS 배지** — 드로어 전용 시각화로, 현재 프로젝트-자산 데이터 모델에 "자산 그룹" 개념이 없음 (자산 그룹핑 데이터 모델 설계 필요)

## 공유 뷰 (`/share/[token]`) — Task 10

- **Overview 3-카드 그리드 중 "Overall Risk Score" 게이지**(예: 88점) — 실제 리스크 스코어 데이터가 없음 (리스크 스코어 산출 로직 및 공유 API 확장 필요)
- **자산 카테고리 그룹 헤더** (예: "Database Servers (3)", "Edge Routers (2)") — 실제 데이터 모델은 `repo`/`server` 2종만 구분 가능, 임의 그룹 생성 방지를 위해 플랫 테이블로 대체 (세부 자산 타입/카테고리 데이터 모델 확장 필요)
- **"View Report" 링크/버튼** (자산 행 hover 시 노출되는 상세 리포트 링크) — 읽기 전용 뷰에 버튼 신설 금지 규칙 + 내부 인증 라우트(`/runs/[id]/report`)로 외부 공유 링크에서 연결하려면 별도 인가 검토 필요 (공유 토큰 기반 리포트 접근 인가 설계 필요)
- **Project Manager 아바타/이메일, Support Contact(SecOps Desk 내선)** — `ShareData`에 이메일·아바타·지원팀 연락처 필드가 없음(의도적으로 `pmEmail` 등이 공유 API에서 제외됨) (공유 API에 노출할 필드 범위를 제품 결정으로 확정 필요)
- **"Read-Only View" 상단 배지, 별도 독립 톱바(사이드바 없는 헤더)** — 현재 앱은 루트 레이아웃에서 이미 사이드바+헤더를 공유 라우트에도 노출 중이며, 브리프가 기존 셸 노출 여부를 유지하도록 명시 (셸 구조를 공유 라우트 전용으로 분리할지는 별도 제품 결정 필요)
- **취약점 판정(양호/취약/검토) 배지** — 공유 API가 파이프라인 상태만 노출하고 있어, outcome(양호/취약) 노출은 별도 제품 결정 필요 (공유 API를 통한 결과 노출 범위 결정 필요)

## 원본 초안 대비 비고

`.superpowers/sdd/backlog.md`(초안)에 있던 항목은 모두 위 목록에 포함되어 있다. 이 문서는 초안에 없던 세부 항목(Task 3의 검색바/토글 버튼, Task 4의 트렌드 화살표·CVSS 게이지 등, Task 6의 취소 버튼·예상 남은 시간·수집률 배지 등, Task 9의 검색창·KPI 벤토·드로어·인라인 액션·토글·Revoke·아코디언, Task 10의 리스크 게이지·카테고리 그룹·PM 연락처·톱바)까지 각 태스크 리포트 전문을 대조해 추가로 반영했다.
