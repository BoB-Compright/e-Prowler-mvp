# QR 참여형 공유 + 모바일 리포트 뷰 설계

> 작성일: 2026-07-14
> 상태: 승인됨(사용자 결정 확정) → 구현 계획 대기

## 목표
발표 시 청중(~20명)이 QR을 스캔해 프로젝트 공유 리포트를 모바일로 보고 참여하게 한다. (1) PM이 공유 링크를
QR 이미지로 띄우고, (2) 청중이 스캔→비밀번호 입력→모바일에서 취약점 리포트를 본다.

## 확정 결정
- **접근 = 생 공유 URL**: QR엔 공유 URL만. 청중이 스캔 후 기존 비밀번호 게이트에 비번 입력(기존 공유 흐름 재사용).
- **QR엔 URL만**(비밀번호는 해시 저장이라 표시 불가). 발표자가 비번을 구두/슬라이드로 안내.
- **QR 표시 = 기존 공유 패널(ShareLinkPanel)에 추가**(기존 디자인 가이드 기반).

## ① QR 표시 (ShareLinkPanel)
- ShareLinkPanel은 이미 `shareUrl = {origin}/share/{token}`을 client에서 보유. 여기에 **"발표용 QR"** 버튼 추가
  (공유 링크 상태 active일 때만). 클릭 시 패널 내 **QR 영역 토글(확장)**.
- **QR 생성**: `qrcode` 라이브러리를 **클라이언트 동적 import**(`await import("qrcode")`)로 최초 열 때만 로드,
  `QRCode.toDataURL(shareUrl, { width: 320, margin: 2 })` → `<img src={dataUrl}>`. data URL이라 CSP 안전(외부 호출 없음).
- 표시: 큰 QR + 공유 URL 텍스트 + **안내 문구**: "청중이 스캔 후 비밀번호를 입력합니다. 비밀번호를 정확히
  안내하세요. 링크가 잠기면 아래 '재발급'으로 복구할 수 있습니다."
- 링크가 active가 아니면(비활성/폐기) QR 버튼 숨김(접근 불가 상태이므로).
- 실패(생성 오류) 시 조용한 에러 문구.

## ② 모바일 리포트 뷰 (ShareGate)
현재도 `grid-cols-1 lg:grid-cols-3`(모바일 스택) + 자산 테이블 `overflow-x-auto`로 부분 반응형. 모바일(≈375px)
정독성 폴리시(구조 변경 없이):
- 페이지 가로 오버플로우 없음 보장(테이블은 자체 `overflow-x-auto` 컨테이너 안에서만 스크롤).
- "조치가 필요한 항목" 카드가 모바일 주 콘텐츠 — 풀폭, 항목 제목·배지 줄바꿈 자연스럽게, 코드블록 가로 스크롤.
- 자산 테이블은 모바일에서 가로 스크롤 유지(핵심은 findings). 필요 시 셀 패딩·글자 크기 모바일 조정.
- 탭 타겟(버튼) 충분한 크기.

## ③ 부하 / 가용성 (판단 — 코드 변경 최소)
- 공유 리포트는 비번 제출 시 **1회 fetch**(폴링 없음) → 청중 20명 동시도 가벼운 일회성 읽기. SQLite 읽기
  순차이나 소량·빠름, 퀵터널 20 동시 여유. 지속 부하 사실상 0. **추가 인프라 변경 불필요.**
- **가용성 리스크 = 공유 링크 잠금**: 청중 비번 오타 누적 시 공유 링크 잠금(전원 차단) 가능. 잠금 정책은
  변경하지 않고, QR 안내 문구 + 발표 중 잠기면 PM이 **재발급(기존 기능)**으로 즉시 복구하도록 안내한다.

## 테스트
- (있으면) QR/URL 순수 로직 단위 테스트. QR 버튼·토글·모바일 폴리시는 tsc/eslint/next build + 수동(모바일 뷰포트).

## 다루지 않는 것
- 비밀번호 없는 발표 토큰, 비번 자동입력, 잠금 정책 변경, QR에 비번 embed, 실시간 참여 집계.
