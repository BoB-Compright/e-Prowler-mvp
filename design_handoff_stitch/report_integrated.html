CRITICAL DB-SEC-042

PostgreSQL 기본 포트 외부 노출 및 인증 우회 가능성

Vulnerability Description (취약점 설명)

데이터베이스 클러스터의 마스터 노드가 0.0.0.0 바인딩을 통해 공인 IP에서 접근 가능한 상태입니다. 또한 `pg_hba.conf` 설정에서 특정 대역에 대해 `trust` 인증을 사용하고 있어, IP 스푸핑을 통한 인증 우회 공격에 취약합니다.

Evidence Log (증거 로그)

# postgresql.conf
listen_addresses = '*'  # 취약: 모든 IP 허용

# pg_hba.conf
host    all             all             10.0.0.0/8            trust  # 취약: 인증 없음

build Remediation (조치 방안)

1. 리슨 주소를 로컬호스트 또는 내부 VPC 대역으로 제한하십시오.
2. `trust` 인증 방식을 `scram-sha-256`으로 변경하여 강력한 패스워드 인증을 강제하십시오. 

# postgresql.conf 조치 후
listen_addresses = 'localhost, 10.10.0.0/16'

# pg_hba.conf 조치 후
host    all             all             10.10.0.0/16          scram-sha-256
HIGH DB-ENC-011

전송 구간 암호화(TLS/SSL) 미적용

Vulnerability Description (취약점 설명)

클라이언트와 데이터베이스 간의 통신 구간에 TLS 암호화가 강제되어 있지 않습니다. 이로 인해 동일 네트워크 내에서 패킷 스니핑을 통한 중요 데이터(쿼리 내용, 결과 값) 유출 위험이 존재합니다.

Evidence Log (증거 로그)

$ psql -h prod-db-cluster -U admin
psql (14.2)
Type "help" for help.

# 연결 상태 확인
SSL connection (protocol: none, cipher: none)  # 취약: SSL 미적용

build Remediation (조치 방안)

데이터베이스 설정에서 SSL을 활성화하고, `pg_hba.conf`에서 클라이언트 연결 시 `hostssl`을 강제하도록 설정하십시오. 유효한 인증서 적용이 필요합니다. 

# postgresql.conf
ssl = on
ssl_cert_file = 'server.crt'
ssl_key_file = 'server.key'

# pg_hba.conf 조치 후 (host 대신 hostssl 사용)
hostssl all             all             0.0.0.0/0             scram-sha-256
